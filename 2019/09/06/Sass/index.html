使用变量

变量声明

为css引入了变量，常用属性封装成变量并反复使用。使用$来标识变量。
声明变量可以用空格或逗号隔开，就像是css那样。{}内部定义的变量只在内部使用，相对的{}之外的可以在各处使用。

变量引用

在编译时相当于将变量“：”后的值代替$variable，且变量值改变则引用值也随之改变。
变量的值也可以引用另一个变量。

$height-color:red
$height-border: 1px solid $height-color

嵌套规则

  • 类似于盒模型，当为一个盒子里的不同元素设置不同样式时，采用嵌套的方法。而解套的方式在这种情况下是后代选择器，即用空格连接。

    #content {

    article {
  h1 { color: #333 }
  p { margin-bottom: 1.4em }
}
aside { background-color: #EEE }

    }
    编译后为：
    ···
    #content article h1 { color: #333 }
    #content article p { margin-bottom: 1.4em }
    #content aside { background-color: #EEE }

  • 而这种方式在遇到伪类选择器如 :hover 等时是不适用的。此时要用特殊标识符“&”来表示父级元素。

    article a {

    color: blue;
&:hover { color: red }   //没有&会导致错误

    }

  • 群组选择器的嵌套，无论是对容器内的群组还是容器本身是群组的情况都适用。

  • 子组合选择器与同层组合选择器： + > ~

  • 嵌套属性 除了选择器外，属性也可以嵌套。

    对于像：background-color 和background 这样以-分隔显示不同属性会有重复的部分。

    导入SASS

    与css原生的@import不同，sass的@import在生成css文件时就把相关文件导入进来，这样就避免了原生@import造成的页面加载速度变慢的后果。

    使用sass的@import规则并不需要指明被导入文件的全名。你可以省略.sass或.scss文件后缀，在不修改样式表的前提下，可以随意修改你或别人写的被导入的sass样式文件语法，在sass和scss语法之间随意切换。

    • 使用sass局部文件的特殊规定，即以下划线开头的文件为局部文件，但@import引入时可以省略下划线。

    如：某局部文件为theme/_partalfile 引入时@import “theme/partalfile”

    • 默认变量值：如果这个变量被声明赋值了，那就用它声明的值，否则就用这个默认值。

    如果用户在导入你的sass局部文件之前声明了一个$fancybox-width变量，那么你的局部文件中对$fancybox-width赋值400px的操作就无效。如果用户没有做这样的声明，则$fancybox-width将默认为400px。

    • 嵌套导入：跟原生的css不同，sass允许@import命令写在css规则内。这种导入方式下，生成对应的css文件时，局部文件会被直接插入到css规则内导入它的地方。

    1. 被导入文件的名字以.css结尾；

这就是说，你不能用sass的@import直接导入一个原始的css文件，因为sass会认为你想用css原生的@import。但是，因为sass的语法完全兼容css，所以你可以把原始的css文件改名为.scss后缀，即可直接导入了。

静默注释

不同于标准css的/**/注释，这种会在生成在css文件中可见（虽然此种注释写在css不允许区域如属性值内部时也会被抹去）。

类似于JavaScript使用” // “注释到行末,其内容不会出现在生成的css文件中。

混合器

独立变量适用于较小的修改，如统一的颜色字体等，对于大段大段的样式不适用，这时可以用@mixin和@include来进行大段样式的复用。

  • 何时使用（避免过度使用混合器造成加载缓慢）

是否能起一个简短、概括起作用的短名。

混合器在某些方面跟css类很像。都是让你给一大段样式命名，所以在选择使用哪个的时候可能会产生疑惑。最重要的区别就是类名是在html文件中应用的，而混合器是在样式表中应用的。这就意味着类名具有语义化含义，而不仅仅是一种展示性的描述：用来描述html元素的含义而不是html元素的外观。而另一方面，混合器是展示性的描述，用来描述一条css规则应用之后会产生怎样的效果。

  • 混合器也可使用嵌套css

    • 混合器传参

      类似于JavaScript的function函数，混合器可以指定传入参数以达到一个混合器产生不同效果。

      sass允许通过语法$name: value的形式指定每个参数的值。这样就解决了顺序问题，但也增加了工作量。

      • 默认参数值

        • 发现某一个class是另一个class的细化时，可以用继承的语法@extend来将多个类整合到一起。
